Networks and HTML

URL

Uniform Resource Locator, also known as the “web address”.

Query

In the context of the World Wide Web, a query is information sent to a server in addition to a URL. Parameters are used for differentiation purposes:

  • ? - Separates the URL from query parameters.
  • & - Separates query parameters from each other.

Port

A port directs a URL to a specific location on a given server. If not shown, then the default port used is 80.

Request Line

This is a command that is sent from a given client to a given server. A command is also known as a “method”. Examples of methods include GET (requesting information from a server) and POST (update information from the user on a web page).

Status Codes

  • 200-Level: Success.
  • 300-Level: Success, but not in a straightforward manner.
  • 400-Level: Client-side (user) error.
  • 500-Level: Server-side error.

Validation

Assurance that data that a user provides is acceptable, either by being within an acceptable range, or by being correctly formatted. This assures that errors will not occur, and that malicious users do not exploit a given website by introducing data that would cause havoc. Validation can be server-side or client-side.
